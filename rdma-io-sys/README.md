# rdma-io-sys

Low-level FFI bindings for [libibverbs](https://github.com/linux-rdma/rdma-core) and [librdmacm](https://github.com/linux-rdma/rdma-core), generated by [bnd](https://github.com/nicira/bnd).

## Modules

| Module | Feature | Contents |
|---|---|---|
| `ibverbs` | `ibverbs` | Bindings for `<infiniband/verbs.h>` — devices, PDs, MRs, CQs, QPs, work requests |
| `rdmacm` | `rdmacm` | Bindings for `<rdma/rdma_cma.h>` — connection management, address resolution |
| `wrapper` | `wrapper` | C wrapper functions for ~93 `ibverbs` inline functions that cannot be linked directly from Rust |

All three features are enabled by default.

## Why the Wrapper?

Many libibverbs functions (e.g., `ibv_post_send`, `ibv_poll_cq`, `ibv_req_notify_cq`, the entire `ibv_wr_*` API) are `static inline` in the C header. Rust's FFI can only link to real symbols, not inline functions. The `wrapper/` directory provides a thin C shim that calls each inline and exports it as a regular symbol:

```c
// wrapper.c
int rdma_wrap_ibv_post_send(struct ibv_qp *qp, struct ibv_send_wr *wr, struct ibv_send_wr **bad_wr) {
    return ibv_post_send(qp, wr, bad_wr);
}
```

The build script (`build.rs`) compiles `wrapper.c` via the `cc` crate.

## Binding Generation

Bindings are generated from a WinMD metadata file (`winmd/bnd-rdma.winmd`) by the `bnd-rdma-gen` crate. The generated source lives in `src/rdma/` and should not be edited manually.

To regenerate:

```sh
cargo run -p bnd-rdma-gen
```

## Prerequisites

```sh
sudo apt install libibverbs-dev librdmacm-dev
```

## Usage

This crate is not intended for direct use. Use [`rdma-io`](../rdma-io/) for a safe, high-level API.

```rust
// Raw FFI (unsafe)
use rdma_io_sys::ibverbs::*;
use rdma_io_sys::wrapper::*;
```
